# Task 4: 更新 secondsight init：生成 ~/.secondsight/config.toml with defaults（含 diff 保護）

## Context

Read: overview.md

`secondsight init` 目前只安裝 Claude Code hooks（`~/.claude/hooks/` + `~/.claude/settings.json`）。本 task 擴充它，在執行時同時生成 `~/.secondsight/config.toml`。

**核心保護規則（D-ST 決策）**：
- config.toml **不存在** → 生成（含所有 built-in default + comment 說明）
- config.toml **已存在且無新 key** → 印出 "config.toml already up-to-date"，不修改
- config.toml **已存在但有新 key**（版本升級新增了 section）→ 印出 diff，**不自動寫入**，提示 operator 手動合併
- config.toml **已存在但 malformed** → 印出錯誤，建議執行 `secondsight config validate`

**`.env` 不生成**：operator 自行管理。Config.toml template 會有 comment 說明如何搭配 `.env` 使用。

**依賴 task-1**：`config/template.py` 的 template 生成函式必須先存在。

## Death Test Requirements

在實作前必須先寫並跑失敗的 death tests：

- **DT-init-1**: `~/.secondsight/config.toml` **已存在**，執行 init → 不覆蓋，檔案內容不變
- **DT-init-2**: config.toml 已存在且 malformed → 不覆蓋，印出 error 訊息，exit 0（不是 1，因為 hook 安裝可以繼續）
- **DT-init-3**: config.toml 不存在 → 生成後，`tomllib.load()` 可以成功解析（template 本身合法 TOML）
- **DT-init-4**: 生成的 config.toml 包含所有 expected sections：`[retention]`, `[analysis]`, `[analysis.models]`, `[analysis.models.fallback]`, `[analysis.read_project_file]`
- **DT-init-5**: config.toml 已存在且有新版本新增的 key → 印出 diff，不寫入，exit 0

    ```toml
    # SecondSight Configuration
    # Generated by: secondsight init
    # 修改此檔案以覆蓋預設值。
    # 使用 secondsight config validate 驗證設定。
    # 使用 ~/.secondsight/.env 存放 secrets（如 ANTHROPIC_API_KEY）。
    # 在此檔案中可用 ${VAR_NAME} 參照 env var。

    [retention]
    # 原始 hook 事件保留天數（預設 90 天）
    raw_traces_ttl_days = 90
    # 分析結果保留天數（預設 365 天）
    analysis_ttl_days = 365
    # 分析完成後是否立即清除原始事件（預設 false）
    cleanup_after_analysis = false

    [analysis]
    # 預設 agent 類型：claude_code | codex | opencode | auto
    # auto = 從最近一次 session 自動偵測
    default_agent = "claude_code"

    [analysis.models]
    # 各 adapter 的 model 覆蓋（空字串 = 使用 adapter 預設值）
    # 可使用 ${VAR_NAME} 參照 env var，例如：claude_code = "${MY_MODEL}"
    claude_code = ""   # 預設: claude-haiku-4-5-20251001
    codex = ""         # 必填，否則分析會失敗
    opencode = ""      # 必填，否則分析會失敗

    [analysis.models.fallback]
    # Primary model 失敗時的 fallback chain（transport error 才觸發）
    fallback_models = ["gpt-4o-mini", "gemini-2.0-flash"]

    [analysis.read_project_file]
    # 是否允許分析 agent 讀取專案檔案（預設啟用）
    enabled = true
    # 單一檔案讀取大小上限（KiB）
    size_cap_kb = 256
    # 額外拒絕讀取的檔案 pattern（疊加在內建 denylist 上）
    denylist = []
    ```
  - `get_template_keys() -> set[str]`：回傳 template 裡所有的 key path（供 diff 用）

## Expected Scar Report Items

- **Scar-1**: `diff_against_template()` 的實作是 key-path 集合比較（flatten TOML doc 成 `section.key` 形式）。這個 flatten 不處理陣列型的 value（例如 `fallback_models` 裡的元素），只比較頂層 key 名稱。如果未來 template 改了某個 key 的型別（string → array），diff 偵測不到，會誤以為 "up-to-date"。記錄在 scar。
- **Scar-2**: `installer.py` 原本的 dry-run 回傳格式是什麼（JSON or text），本 task 的 config.toml 生成訊息需要跟它一致。實作前先確認 `installer.py` 的輸出格式，不要產生風格不一致的 CLI 輸出。
